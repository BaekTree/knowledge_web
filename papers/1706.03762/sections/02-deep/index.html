2 Figure 1 The Transformer - model architecture

The Transformer replaces recurrence with a stack of identical layers built from self-attention and position-wise feed-forward networks. In the encoder, each of the six layers has two sub-layers: multi-head self-attention, which lets each token look at other tokens in the input, and a fully connected feed-forward network applied independently to each position. Both sub-layers use residual connections plus layer normalization, written as `LayerNorm(x + Sublayer(x))`, to stabilize training and help gradients flow through a deep stack. All layer outputs use the same model dimension, `d_model = 512`, so information can move cleanly between components. The decoder follows the same basic pattern but adds a third sub-layer that attends to the encoder’s output, allowing it to condition generation on the encoded input sequence. Its self-attention is masked so a position can only use earlier positions, not future ones. That masking, together with shifting the target sequence by one position, enforces the autoregressive rule: when predicting token `i`, the model only sees tokens `< i`. This design makes the Transformer behave like a proper sequence generator while still enabling parallel computation during training. Section 3.2 then formalizes attention itself as a mapping from queries and key-value pairs to an output vector. The motivation is simple: rather than compressing all context into a single hidden state, attention builds a weighted combination of values based on relevance between a query and each key. This is the core mechanism that powers both self-attention in the encoder/decoder and the encoder-decoder attention in the decoder. The next part of the paper uses this definition to derive scaled dot-product attention and multi-head attention, which are the specific implementations that make the architecture work in practice.
