4 Why Self-Attention

This section motivates self-attention by comparing it with recurrent and convolutional layers along three axes: total computation per layer, how much can be parallelized, and how long the “information path” is between two positions in a sequence. The basic idea is that sequence models must transform an input sequence \((x_1, \dots, x_n)\) into an output sequence \((z_1, \dots, z_n)\), and the easier it is for a signal to move between distant tokens, the easier it is to learn long-range dependencies such as subject-verb agreement or coreference. Self-attention is attractive because every position can directly attend to every other position in a constant number of sequential steps. The paper’s comparison is not just qualitative; it is a complexity argument. A self-attention layer has only \(O(1)\) sequential depth and can be faster than recurrence when the sequence length \(n\) is smaller than the representation size \(d\), which is common in machine translation. Recurrent layers, by contrast, require \(O(n)\) sequential operations because they process tokens one after another. Convolutions improve parallelism over recurrence, but a kernel of width \(k\) does not connect all positions in one layer, so multiple stacked layers are needed to propagate information globally. That increases the longest path between positions, which can make learning harder. The authors also note variants such as local attention and separable convolutions, but present them mainly as tradeoffs rather than their final design. This section sets up the model architecture in the next part of the paper. Having argued that self-attention offers shorter paths, better parallelism, and competitive complexity, the authors can justify building the Transformer from attention plus feed-forward blocks rather than recurrence or standard convolution. The final point about interpretability also foreshadows later discussion of attention heads, where different heads specialize in different syntactic and semantic relations.
