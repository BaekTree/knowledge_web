Self-Harness: Harnesses That Improve Themselves
arxiv:2606.09498

Section 9 deeper explanation

Abstract

This abstract introduces two coupled pieces of an LLM agent: the base model, which produces actions, and the harness, which is the surrounding control logic that formats prompts, interprets outputs, retries failures, and interfaces with the environment. The key idea is that performance depends on both, and that harnesses are not one-size-fits-all because different models fail in different ways. Self-Harness is the paper’s proposal for closing this gap: instead of having humans hand-tune the harness, the model itself iteratively improves its own operating wrapper. The method is organized as a three-stage loop. First, Weakness Mining scans execution traces, which are records of what the agent tried and where it failed, to identify recurring model-specific failure patterns. Second, Harness Proposal turns those weaknesses into candidate harness edits, but keeps the edits “minimal” so the system changes only what is necessary rather than rewriting the whole setup. Third, Proposal Validation tests each candidate against regressions, meaning it only accepts changes that help on the targeted failures without breaking existing behavior. That design choice matters because harnesses are easy to overfit: a clever patch can fix one benchmark and silently damage generality. The results section is foreshadowed here by showing that the approach is instantiated on Terminal-Bench-2.0 with a deliberately small initial harness and three diverse models. The reported gains suggest the loop is not just adding generic prompting tricks; it is converting concrete failure traces into executable control changes. This sets up the next part of the paper, which will need to show how the loop works in detail and why these harness edits reliably improve performance across model families.
