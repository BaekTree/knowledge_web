Experiments

The Experiments section asks a narrow but important causal question: can the system improve by changing only the harness, while keeping the underlying language model fixed? That isolates the contribution of the orchestration layer from model capability itself. The evaluation is run on Terminal-Bench-2.0, a benchmark for terminal-based task completion in containerized environments, so success depends on the agent’s interaction loop, tool use, and error recovery rather than on pure text generation. This makes it a good testbed for Self-Harness, because many failures in agentic systems come from brittle scaffolding around the model, not from the model’s raw reasoning alone.

The experimental setup starts from the same minimal DeepAgent-based harness for every backend model. “Minimal” matters here: it gives a low-complexity baseline with few hand-engineered advantages, so any gains can be attributed to Self-Harness rather than to a rich initial prompt or elaborate control logic. Self-Harness then runs an iterative loop in which the harness proposes edits, validates them using held-in execution evidence, and promotes them only if they pass held-out regression gates. The design reflects a standard engineering tradeoff: you want to explore changes aggressively enough to improve performance, but constrain them tightly enough that a local fix does not silently break other tasks.

The key technical idea is bounded edit search under evidence-based selection. Instead of rewriting the entire agent, Self-Harness makes small, localized changes to the harness behavior and checks them against two kinds of feedback. Held-in execution evidence supplies the immediate signal that a candidate edit helps on the examples currently being observed. Held-out regression gates act like a guardrail: they test whether the edit generalizes beyond the specific cases that motivated it. Intuitively, this is the same logic as training and validation splits in supervised learning, but applied to agent control policies and runtime scaffolding rather than to model weights. The “formula” behind the process is essentially a constrained optimization loop: maximize task success while keeping regressions below an acceptable threshold.

This section connects directly to the paper’s broader claims. In the Benchmarks and Splits/protocol framing, the authors need a setup where improvement can be attributed cleanly and measured reliably. In the Models framing, they deliberately hold the model constant to show that harness quality is itself a major lever. And in the Main Results, these experiments provide the evidence that bounded harness edits can produce real gains, not just one-off wins on a narrow prompt set. The broader implication is that agent performance is a systems problem: better orchestration can unlock more of a fixed model’s capability without changing the model at all.
