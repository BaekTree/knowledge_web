Self-Harness: Harnesses That Improve Themselves
arxiv:2606.09498

Main Results

This section introduces the core evaluation idea behind Self-Harness: the harness is not a fixed wrapper, but something that can be edited and then “promoted” if the edits improve benchmark performance. The authors measure success on Terminal-Bench-2.0 using Pass(%), which is simply the fraction of tasks solved. They test three different model backends and report results on two splits: held-in, the failures used to generate proposal evidence, and held-out, a separate split used to check whether the edits generalize beyond the examples that motivated them. A key design choice is to report both absolute gains in percentage points and relative gains. Absolute gain shows the direct change in Pass(%), while relative gain normalizes that change by the starting score, which matters when comparing models that begin at very different baselines. For example, Qwen3.5 starts much lower than the others, so a similar-sized absolute improvement corresponds to a much larger relative jump. This makes it easier to see whether the harness edits help mainly by patching easy failures or by improving the execution mechanism in a more reusable way. The main technical claim is that harness-level changes can improve performance while keeping everything else fixed: the backend model, tool set, budget, benchmark environment, and evaluator do not change. That isolates the harness as the source of improvement. The fact that all three models improve on held-out tasks is especially important, because it shows the edits are not just memorizing the failure cases used to propose them. The regression gate then acts as a safeguard, ensuring a candidate edit only gets promoted if it does not harm either split. This result sets up the next stage of the paper by justifying the promotion mechanism itself. Once the authors establish that harness edits can generalize and avoid regressions, the natural next question is what kinds of edits were made, why they work, and how the self-improvement loop selects them. In other words, this section provides the empirical evidence that the Self-Harness pipeline is worth studying in more detail.
