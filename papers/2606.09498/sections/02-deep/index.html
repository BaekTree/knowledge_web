Introduction

This introduction frames an LLM agent as more than just a base model: its behavior is shaped by the harness around it, meaning the system prompt, tools, orchestration logic, verification rules, and recovery behavior. The key technical idea is that the same model can perform very differently under different harnesses, so improving agent performance is not only a model problem but also a systems problem. The paper proposes “Self-Harness,” where the target agent improves its own harness instead of relying on humans or a stronger external model. That choice matters because model-specific failures are often systematic, and a harness tuned for one model may be a poor fit for another. The method is organized as an iterative improvement loop with three stages. First, Weakness Mining runs the current harness on tasks and clusters failed execution traces, so the agent can identify recurring failure patterns rather than isolated mistakes. Second, Harness Proposal asks the agent to generate small, diverse edits that each address a specific failure mode; keeping edits minimal is a deliberate design choice to avoid broad, noisy changes that are hard to attribute. Third, Proposal Validation tests each candidate with regression checks and only keeps changes that improve performance without harming held-out tasks. This makes the process conservative: it favors edits that generalize rather than overfitting to the observed failures. The experiments show why this loop is useful. Across three different base models, Self-Harness improves both held-in and held-out performance on Terminal-Bench-2.0, which suggests the harness changes are not just memorizing evaluation traces. The qualitative examples reinforce the mechanism: different models benefit from different fixes, such as earlier file creation, better dependency checking, or preserving shell state. The next part of the paper would naturally build on this setup by specifying the exact harness-update process, since the introduction has established the problem, the loop, and the empirical motivation.
