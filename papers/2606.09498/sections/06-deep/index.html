Self-Harness: Harnesses That Improve Themselves
arxiv:2606.09498

Preliminary

This section introduces the paper’s central abstraction: a *harness* is the non-parametric layer around a fixed language model that determines how it is used as an agent. In this setup, the model weights never change. Instead, the harness controls the instruction prompt, tool access, memory, state handling, and the execution protocol for interacting with a task. That makes the harness the thing being optimized: the same model can behave very differently depending on how it is deployed. The formal notation reflects this separation by treating the model `M` and evaluator `E` as fixed, while the harness `h` is the object updated across iterations. The algorithm then defines a search process over harness versions `h_0, h_1, ..., h_T`. For each round, the current harness is evaluated on both a held-in split `D_in` and a held-out split `D_ho`, producing performance metrics and execution traces. Those traces are used to build an evidence bundle from verifier-grounded failures, meaning the system focuses on cases where the harness demonstrably broke down rather than on vague errors. The model is then asked to propose multiple candidate edits in parallel, which increases exploration without committing to a single guessed fix. Each proposal is turned into a new harness, evaluated again, and accepted only if it improves held-in performance without hurting held-out performance. This is a deliberate anti-overfitting rule: it rewards genuine protocol improvements, not edits that merely memorize the training split. If no candidate passes, the harness stays unchanged; otherwise, the accepted edits are merged into the next harness version. Conceptually, this is an iterative self-improvement loop, but at the level of execution scaffolding rather than model parameters. The next part of the paper builds on this foundation by showing how the evidence bundle, proposal mechanism, and acceptance criterion are instantiated in practice, and how these harness edits are structured so they can reliably improve agent behavior.
