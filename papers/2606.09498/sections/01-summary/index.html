Section 1 summary

Abstract

This abstract frames the paper’s core claim: agent performance depends not only on the base model but also on the harness, and that harness can be improved by the model itself. It introduces Self-Harness as an iterative process with three parts: mining model-specific failures from traces, proposing minimal harness edits that target those failures, and validating those edits through regression tests. It also summarizes the main empirical result that this loop improves several different base models on Terminal-Bench-2.0, showing the idea works across model families. The section exists to answer the paper’s central question up front: can an LLM-based agent systematically reshape its own operating harness, rather than relying on humans or stronger external agents, and does that actually produce better performance?
