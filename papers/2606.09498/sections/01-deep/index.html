Self-Harness: Harnesses That Improve Themselves
arxiv:2606.09498

Section 1 deeper explanation

Abstract

This abstract introduces a key idea: an agent’s behavior is not determined only by the base LLM, but also by the “harness,” meaning the surrounding code, prompts, tools, and control logic that mediate how the model acts in an environment. The authors argue that harnesses must be model-specific because different LLMs fail in different ways, yet today they are still hand-designed by humans. That becomes a bottleneck as models diversify quickly. Self-Harness is the proposed alternative: instead of relying on human engineers or a stronger outside agent, the model improves the harness it runs inside. Technically, Self-Harness is an iterative feedback loop with three stages. Weakness Mining analyzes execution traces to find recurring failure patterns, such as where a model consistently misuses tools or gets stuck in a particular interaction style. Harness Proposal then turns those mined weaknesses into candidate harness edits, but keeps the changes minimal and diverse so the system does not drift into broad, hard-to-attribute rewrites. Proposal Validation acts as a safeguard: a candidate change is accepted only if regression tests show it helps without breaking previously working behavior. That design choice matters because harness changes can easily overfit to one failure mode while degrading overall performance. The abstract also previews the empirical result that motivates the rest of the paper: when this loop is instantiated on Terminal-Bench-2.0 with a minimal starting harness, all three tested models improve substantially. The specific pass-rate gains show that the method is not tied to one architecture or one model family. The final qualitative claim sets up the next sections: the improvement is not just from adding generic instructions, but from converting concrete weaknesses into executable harness modifications. That naturally leads into the paper’s methods and analysis of how the loop actually mines failures, proposes edits, and validates them.
