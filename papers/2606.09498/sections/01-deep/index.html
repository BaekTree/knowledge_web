Self-Harness: Harnesses That Improve Themselves
arxiv:2606.09498

Section 1 deeper explanation

Abstract

This abstract argues that an agent’s behavior is not determined only by the base LLM, but also by the “harness” around it: the code, prompts, tools, and control logic that mediate interaction with the environment. The key technical idea is that harness design is model-specific, because different LLMs fail in different ways. Self-Harness is the proposed remedy: instead of humans hand-authoring better harnesses, the model itself iteratively improves its own harness without stronger external agents. That turns harness design into a closed-loop optimization problem. The method is organized into three stages. First, Weakness Mining analyzes execution traces to find recurring, model-specific failure patterns. Second, Harness Proposal generates candidate harness edits that are both diverse and minimal, meaning they target the observed weakness without introducing unnecessary changes that could overfit or destabilize behavior. Third, Proposal Validation filters edits through regression testing, accepting only those that fix the target issue while preserving earlier capabilities. This validation step is important because harness changes can easily create new failures elsewhere; the paper’s design explicitly trades off improvement against regressions. The experimental setup instantiates this loop on Terminal-Bench-2.0 with a very small starting harness and three different base models. The reported gains show that the approach generalizes across model families rather than being tuned to one architecture. The section sets up the next part of the paper by establishing the core pipeline and the evidence that self-directed harness revision can produce concrete performance gains, motivating deeper analysis of what kinds of failures are discovered and what edits the system learns to make.
