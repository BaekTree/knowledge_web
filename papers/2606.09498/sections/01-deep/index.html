Abstract

The abstract’s central claim is that agent performance is not determined by the language model alone. It depends jointly on the base model and the harness: the surrounding code, prompts, tools, and control logic that decide how the model is queried, how actions are executed, and how failures are handled. That matters because different models have different strengths and failure modes, so a harness that works well for one model may be suboptimal for another. The paper argues that this makes harness design inherently model-specific, but in current practice harnesses are usually hand-built by humans, which does not scale as models change quickly and proliferate.

Self-Harness is the proposed alternative: instead of relying on human engineers or a stronger external model, an agent improves its own harness through an iterative loop. The loop has three stages. Weakness Mining examines execution traces to find recurring, model-specific failure patterns. Harness Proposal then converts those failures into concrete harness edits, but aims to keep the edits minimal and diverse rather than making broad, generic changes. Proposal Validation filters those candidate edits through regression testing so only changes that help without breaking existing behavior are kept. This design is important because it treats harness improvement as a controlled search problem, not as free-form prompt rewriting.

The abstract’s results section gives the headline evidence that this is worthwhile. On Terminal-Bench-2.0, using a minimal initial harness, Self-Harness improves held-out pass rates for all three tested models: MiniMax M2.5 rises from 40.5% to 61.9%, Qwen3.5-35B-A3B from 23.8% to 38.1%, and GLM-5 from 42.9% to 57.1%. These are substantial gains, especially because they come from the same self-improvement procedure applied across different model families. That cross-model consistency supports the paper’s claim that the method is general in spirit but still produces model-specific harness changes.

A key interpretive point is that the improvements are not just “prompt padding” or adding generic instructions. The qualitative analysis reported in the abstract says the system turns observed weaknesses into executable harness modifications tailored to those weaknesses. That distinction connects directly to the paper’s broader framing in the Introduction and the later sections on the iterative loop, experiments, and main results: the contribution is a method for making harnesses adaptive, where the harness becomes part of the agent’s learning process rather than a fixed artifact designed once by humans.
