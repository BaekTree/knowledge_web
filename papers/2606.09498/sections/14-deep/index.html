Self-Harness: Harnesses That Improve Themselves
arxiv:2606.09498

Section 14 deeper explanation

Preliminary

This section defines the central object of the paper: the harness. A harness is the non-parametric scaffolding around a fixed language model, including instructions, tools, memory, state tracking, and the protocol for when the model can observe, act, verify, and answer. The key point is that the model weights do not change. Instead, the system behavior changes by editing the execution setup that surrounds the model. Formally, the paper treats the model `M` and evaluator `E` as fixed, and studies a sequence of harnesses `h_0, h_1, ..., h_T`, where each new harness is produced by a bounded edit to the previous one. The algorithm, Self-Harness, is a search-and-selection loop over harness edits. At each round, the current harness is evaluated on a held-in split `D_in` and a held-out split `D_ho`, producing performance metrics and traces. Those traces are distilled into an evidence bundle, specifically from held-in verifier-grounded failures, so the proposal process is informed by concrete breakdowns rather than vague intuition. The model then proposes `K` candidate edits in parallel, which makes exploration broader and cheaper than serially testing one idea at a time. Each candidate is applied to the current harness and re-evaluated on both splits. The acceptance rule is deliberately strict: a candidate is only accepted if it improves the held-in metric, does not hurt held-out performance, and satisfies an additional positive-score condition. This is meant to prevent overfitting to the training-like split while still allowing local improvements. If multiple candidates are accepted, they are merged into the next harness; if none are accepted, the harness stays unchanged. The section sets up the rest of the paper by establishing the optimization target and the iterative improvement loop, which later sections can then use to show how harness edits are proposed, validated, and accumulated into better agent behavior.
