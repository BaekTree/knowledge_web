Sites

ClawBench's core innovation is targeting state-changing web tasks—the operations users actually perform online—evaluated against real websites rather than sandboxed simulators. Unlike prior benchmarks that use offline recordings or synthetic environments, ClawBench captures objective ground truth through HTTP request interception: every form submission, purchase attempt, or application generates a measurable payload that agents must produce correctly. This design choice reflects a critical insight: testing whether an agent can interact with the web requires observing what the web actually receives, not just what the agent intended to do. By requiring tasks to be completable today (filtering for paid services and geographically restricted sites), ClawBench prioritizes ecological validity—testing agents against authentic complexity they'll encounter in deployment—over artificial reproducibility.

The benchmark is grounded in human-annotated reference trajectories. Every one of the 153 tasks was executed end-to-end by human annotators who recorded both how they completed the task (the sequence of clicks, form fills, and navigation) and what they submitted to the server (the exact HTTP request body). This dual capture serves two purposes: it creates a ground-truth target for evaluation, and it enables the interception mechanism itself—by logging the expected submission specification, the Chrome DevTools Protocol can reliably distinguish the terminal submission from intermediate HTTP noise. The categorization into eight groups and fifteen fine-grained categories ensures coverage across realistic use-cases: booking travel, managing finances, applying for jobs, social interactions, and academic tasks.

The data capture layer is deceptively sophisticated: beyond recording the intercepted submission, ClawBench generates five synchronized behavioral channels for every agent run—video, per-step screenshots, full HTTP logs with timing, reasoning traces in JSON, and low-level events like keystrokes and scrolls. This redundancy is intentional. The screenshots enable human review and side-by-side comparison with reference trajectories; the HTTP logs preserve what the server saw (independent of agent claims); the reasoning traces reveal agent decision-making. By blocking requests at the HTTP level after logging, ClawBench observes real-world website complexity—how sites respond to partially-filled forms, what errors they return, how they handle concurrent requests—without causing financial charges or permanently altering accounts. It's a form of "ecological validity under sandbox constraints": agents navigate authentic websites but with a safety valve that prevents permanent side effects.

Evaluation uses an agentic evaluator—a Claude Code instance following a fixed rubric—to perform step-level alignment: mapping agent actions to reference actions, detecting divergences, and verifying that required fields and actions were correct. The success metric is formalized as \( \text{Success Rate} = \frac{1}{|T|} \sum_t \text{Score}(t) \), where \( \text{Score}(t) \in \{0, 1\} \) for each task—a simple binary: did the agent produce a submission payload matching the reference? The formula's bluntness is a deliberate design choice. Rather than partial credit for "almost correct" submissions, ClawBench imposes binary judgment, because in real applications, a form submission that's 95% correct still fails authentication or payment processing. However, the rubric includes pragmatic rules: if all prior agent actions are correct and the failure is due to the interception system or verification blocks, the task scores as PASS. This prevents penalizing agents for infrastructure constraints beyond their control, focusing measurement on decision-making rather than environmental brittleness.

This evaluation infrastructure directly enables the benchmark's comparative mission: by capturing complete trajectories and applying consistent scoring, ClawBench can rank model capabilities across task categories and identify systematic weaknesses. The results—Claude Sonnet 4.6 at 33.3% vs. GPT-5.4 at 6.5%—reveal that current models struggle with compound web tasks, and that performance varies sharply across categories, suggesting future work should focus on specific affordances (e-commerce flows, form-filling in novel interfaces, sequential decision-making under uncertainty) rather than uniform improvements.
