Evaluation Methods for Agent Systems

Evaluating web agents is hard because success is not just a final answer, but a sequence of actions that may vary widely across runs. Traditional action-level metrics compare an agent’s trajectory to a reference trace, but that breaks down when there are multiple valid ways to finish the same task. Final-state evaluation avoids that specific problem by checking only whether the end condition was reached, but for everyday web tasks it can be too weak: an agent might appear successful while using the wrong form fields, skipping required confirmations, or relying on brittle side effects. This section frames the core tension in benchmark design: the more a checker ignores the process, the easier it is to automate, but the less trustworthy it becomes for consequential tasks. The three common alternatives each solve part of the problem and introduce new ones. Scripted checkers, including URL-based rules and deterministic state checks, are attractive because they are reproducible and cheap, but they only work when the task outcome can be encoded cleanly in advance. Screenshot comparison is more flexible, but it depends on similarity thresholds that are inherently noisy and can vary with rendering details. Human judgment handles ambiguity well, yet it is expensive, slow, and difficult to reproduce consistently across annotators and time. The paper’s point is not that these methods are useless, but that none of them alone gives the right balance of scalability, robustness, and fidelity for real web-agent evaluation. ClawBench’s answer is to make the evaluation trajectory-aware and evidence-grounded. Instead of judging only the final state or loosely comparing pixels, it intercepts submission payloads and asks an Agentic Evaluator to align the agent’s actions step by step against a human reference trajectory. The reason for this design is that many online tasks have hidden constraints: the correct outcome depends on whether the agent selected the right target, submitted the right payload, and followed the required sequence, not just whether a page eventually looks plausible. The structured justification matters because it ties the verdict to recorded evidence, which makes the result easier to audit and less arbitrary than a free-form LLM judgment. This places the evaluator in the broader benchmark architecture described elsewhere in the paper. The Agentic Evaluator is not the task performer but the scoring mechanism: it consumes the trajectory, compares it to the reference, and emits a binary pass/fail judgment plus rationale. That fits the Scoring discussion because the benchmark needs a repeatable rule for success, and it fits the Benchmark framing because the goal is to standardize evaluation across many heterogeneous web tasks. In the broader landscape of concurrent and complementary work, ClawBench occupies a middle ground between brittle scripted checks and fully subjective LLM-as-judge approaches: it uses agentic reasoning, but constrains that reasoning to evidence already captured during execution.
