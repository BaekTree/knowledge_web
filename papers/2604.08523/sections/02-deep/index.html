Introduction

The introduction is making a precise claim about the current gap between impressive agent demos and dependable general-purpose online assistance. Today’s LLM-based agents can already click through interfaces, type into forms, and chain together multiple actions, but that does not imply they can robustly complete the everyday tasks people actually rely on. The hard part is not simple navigation; it is surviving the messiness of live websites, where pages change, login flows appear, anti-bot systems intervene, and the interface often mutates between runs. The paper argues that existing benchmarks mostly avoid these realities by using offline sandboxes with static HTML and simplified interaction patterns, so they measure competence in a cleaned-up environment rather than true end-to-end task completion. The core technical design choice is to keep the task on production websites while still making evaluation safe enough to run at scale. Instead of cloning sites into a sandbox, ClawBench lets the agent operate on the live web, then reduces risk with an interception mechanism: a Chrome extension records browser-level actions, and a CDP-based server watches outgoing network traffic and blocks the final submission request that would actually commit a transaction. The intuition is that most of the task can be observed and scored without allowing the irreversible last step to go through. That preserves ecological validity, because the agent still faces real authentication, dynamic content, and real site behavior, but it prevents accidental purchases, submissions, or other harmful side effects. To support evaluation, the benchmark captures five aligned layers of evidence for every task: a session replay, per-step screenshots, HTTP traffic, agent messages, and low-level browser actions such as clicks and keystrokes. This is more than logging for convenience. It creates a trace that can be compared against human-completed reference trajectories, which matters because many web tasks have multiple superficially plausible action sequences. A final-state-only score would miss whether the agent reached the right page for the wrong reasons, skipped required steps, or formed the wrong payload. The five-layer recording infrastructure gives the evaluator enough context to diagnose where the trajectory diverged. The evaluation protocol is also deliberately agentic rather than purely programmatic. A Claude Code sub-agent applies a fixed rubric over the task instruction, human reference, and agent trajectory, then returns a binary judgment plus structured justification. The key idea is to evaluate whether the agent completed the task correctly in the sense that a human reference trajectory establishes, not merely whether a final page happened to look right. This setup ties directly to the paper’s broader contribution stack: the benchmark design defines the problem, the interception mechanism makes live-site testing safe, the recording infrastructure makes the behavior auditable, and the evaluator turns that evidence into traceable pass/fail outcomes. The reported main results then show why this setup matters: strong agents on older web benchmarks can still fail sharply on ClawBench, revealing a real gap between demo-quality web interaction and reliable everyday assistance.
